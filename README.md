# NeuroScript

**Neurophysiologically Adaptive Literary Writing System**

NeuroScript is a full-stack web application that uses real-time EEG signals from a Muse headband to dynamically modulate the literary style of text generated by an LLM. The system reads your brainwaves, computes a neurophysiological state vector (calm, activation, fatigue, signal quality), and uses it to shape the form of the prose — sentence length, rhythm, sensory density, verb tense, coherence, and ellipsis — in real time.

> The content stays on topic. The **form** changes with your brain.

## How It Works

```
┌──────────────┐     ┌──────────────────┐     ┌───────────────────┐
│  Muse EEG    │────▶│  Signal Processor │────▶│  Literary Engine   │
│  (or Simulator)    │  Band Power → 4D  │     │  State → Style     │
└──────────────┘     │  NeuroState Vector │     │  Prompt Builder    │
                     └──────────────────┘     └─────────┬─────────┘
                                                        │
                                                        ▼
┌──────────────┐     ┌──────────────────┐     ┌───────────────────┐
│  Frontend    │◀────│  WebSocket Server │◀────│  LLM (GLM-4.7)    │
│  Next.js UI  │     │  FastAPI          │     │  Streaming Text    │
└──────────────┘     └──────────────────┘     └───────────────────┘
```

1. **EEG Input** — Raw EEG data from a Muse headband (or the built-in simulator) is captured at 256 Hz across 4 channels (TP9, AF7, AF8, TP10).

2. **Signal Processing** — Band power is extracted via Welch's method (scipy) for delta, theta, alpha, beta, and gamma bands. These are mapped to a **4D neurophysiological state vector**:
   - **Calm** (0–1): Derived primarily from alpha dominance
   - **Activation** (0–1): Derived from beta + gamma activity
   - **Fatigue** (0–1): Derived from theta + delta activity
   - **Quality** (0–1): Signal-to-noise estimate

3. **Literary Engine** — The 4D vector is translated into concrete literary transformation instructions:
   - High calm → long flowing sentences, sustained rhythm, present continuous tense
   - High activation → short fragmented sentences, staccato rhythm, imperative tense
   - High fatigue → reduced coherence, associative drift, frequent ellipsis
   - Low signal quality → repetitions and syntax contamination

4. **LLM Generation** — The style instructions are injected into the system prompt sent to ZhipuAI GLM-4.7-flash (streaming). The LLM generates prose that embodies the neurophysiological state through form, not content.

5. **Frontend** — A real-time dashboard shows the EEG state, active transformations, and the generated text with a typewriter effect whose speed is adjustable.

## Architecture

| Layer | Technology | Purpose |
|-------|-----------|---------|
| EEG Source | Muse headband via LSL / Built-in simulator | Raw EEG data |
| Signal Processing | NumPy, SciPy (Welch PSD) | Band power → 4D state vector |
| Literary Engine | Python (rule-based mapping) | State → style prompt instructions |
| LLM | ZhipuAI GLM-4.7-flash (OpenAI-compatible API) | Streaming text generation |
| Backend Server | FastAPI + WebSocket | Real-time bidirectional communication |
| Frontend | Next.js 16, React 19, TailwindCSS 4, shadcn/ui | Dashboard UI |
| Charts | Recharts | Temporal EEG state visualization |
| TTS | Web Speech API | Text-to-speech with play/pause/stop |

## Installation

### Backend Setup

```bash
# 1. Navigate to the backend directory
cd backend

# 2. Create and activate a Python virtual environment
python3 -m venv .venv
source .venv/bin/activate        # macOS/Linux
# .venv\Scripts\activate         # Windows

# 3. Install Python dependencies
pip install -r requirements.txt

# 4. Create the environment file
cp .env.example .env  # or create it manually (see below)
```

Create `backend/.env` with your API key:

```env
ZHIPUAI_API_KEY=your_api_key_here
ZHIPUAI_MODEL=glm-4.7-flash
ZHIPUAI_BASE_URL=https://open.bigmodel.cn/api/paas/v4
EEG_SOURCE=simulator
```

| Variable | Description | Default |
|----------|------------|---------|
| `ZHIPUAI_API_KEY` | Your ZhipuAI API key (required) | — |
| `ZHIPUAI_MODEL` | Model name | `glm-4.7-flash` |
| `ZHIPUAI_BASE_URL` | API base URL | `https://open.bigmodel.cn/api/paas/v4` |
| `EEG_SOURCE` | `simulator` (no hardware) or `muse` (real headband) | `simulator` |

### Frontend Setup

```bash
# 1. Navigate to the frontend directory
cd frontend

# 2. Install Node.js dependencies
npm install

# 3. (Optional) Verify the build
npm run build
```

## Configuration

All backend settings are centralized in `backend/config.py` using pydantic-settings. Key parameters:

| Parameter | Value | Description |
|-----------|-------|-------------|
| `eeg_sample_rate` | 256 Hz | Muse headband sample rate |
| `eeg_n_channels` | 4 | TP9, AF7, AF8, TP10 |
| `eeg_update_interval` | 0.5s | State vector update frequency |
| `eeg_buffer_seconds` | 5s | Window for band power analysis |
| `generation_interval` | 12.0s | Seconds between paragraph generations |
| `smoothing_factor` | 0.3 | EMA smoothing for state vector |
| `cors_origins` | `localhost:3000` | Allowed frontend origins |

## Running the Application

You need **two terminals** — one for the backend, one for the frontend.

### Terminal 1: Backend

```bash
cd backend
source .venv/bin/activate
python main.py
```

The backend starts at **http://localhost:8000**. You can verify it's running:

```bash
# Health check
curl http://localhost:8000/api/health
# → {"status":"ok","model":"glm-4.7-flash","eeg_source":"simulator"}

# LLM connection test
curl http://localhost:8000/api/test-llm
# → {"status":"ok","response":"NeuroScript connected","model":"glm-4.7-flash"}
```

### Terminal 2: Frontend

```bash
cd frontend
npm run dev
```

The frontend starts at **http://localhost:3000**. Open it in your browser.

## Using NeuroScript

1. **Configure** — In the left panel, set your theme/scene (e.g., "a detective interrogating a suspect in a dim room"), choose Generator or Editor mode, and adjust sensitivity.

2. **Apply Configuration** — Click the gradient "Apply Configuration" button.

3. **Start** — Click "Start" in the center panel. The system begins:
   - The EEG simulator (or real Muse) streams brainwave data
   - The right panel shows live neurophysiological state bars and a temporal chart
   - Every ~12 seconds, a new paragraph is generated with style shaped by the current brain state
   - Active literary transformations are displayed in real time

4. **Writing Speed** — Use the slider between the text area and controls to adjust how fast words appear:
   - *Instant* — text appears as received from the LLM
   - *Fast / Slow / Pensive / Meditative* — words drip one at a time, up to 3 seconds per word

5. **Text-to-Speech** — Click the speaker icon to have the generated text read aloud. Pause or stop at any time.

6. **Save / Copy** — Use the purple "Save" button to download the session text as a `.txt` file, or the copy icon to copy to clipboard.

7. **Pause / Resume / Stop** — Control the generation session at any time.

## Using a Real Muse Headband

To use a real Muse headband instead of the simulator:

### 1. Install Muse dependencies

```bash
cd backend
source .venv/bin/activate
pip install muselsl pylsl
```

### 2. Start the Muse LSL stream

Make sure your Muse headband is paired via Bluetooth, then:

```bash
muselsl stream
```

This creates an LSL (Lab Streaming Layer) stream that the backend can read.

### 3. Switch the EEG source

Edit `backend/.env`:

```env
EEG_SOURCE=muse
```

### 4. Restart the backend

```bash
python main.py
```

The backend will automatically search for and connect to the Muse LSL stream.

## Tech Stack

### Backend
- **Python 3.10+**
- **FastAPI** — Async web framework with WebSocket support
- **uvicorn** — ASGI server
- **httpx** — Async HTTP client for LLM API calls
- **NumPy** — Numerical computation
- **SciPy** — Welch PSD for EEG band power extraction
- **pydantic-settings** — Type-safe configuration from environment variables
- **ZhipuAI GLM-4.7-flash** — Large language model (OpenAI-compatible API)

### Frontend
- **Next.js 16** — React framework with App Router
- **React 19** — UI library
- **TypeScript 5** — Type safety
- **TailwindCSS 4** — Utility-first CSS
- **shadcn/ui** — Accessible component primitives (Button, Slider, Card, Badge, ScrollArea)
- **Recharts** — Responsive charts for EEG temporal data
- **Lucide React** — Icon library
- **Web Speech API** — Browser-native text-to-speech

### Communication
- **WebSocket** — Real-time bidirectional streaming between backend and frontend
- **JSON messages** — Typed protocol for EEG state, transformations, text chunks, and control commands
